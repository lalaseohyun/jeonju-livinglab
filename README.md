# 전주 영화의 거리 리빙랩 — 만족도조사 웹앱

> **프로젝트 인계용 README** — 클로드 코드가 이 파일을 먼저 읽으면 프로젝트 전체 맥락을 파악할 수 있습니다.

---

## 🎬 프로젝트 개요

**목적**: 전주 영화의 거리 리빙랩 프로젝트 성과 전시·홍보 부스에서 사용하는 실시간 만족도조사 웹앱

**참여자 동선**:
1. 부스 방문 → 태블릿 또는 본인 폰(QR)으로 응답 (2~3분)
2. 응답 직후 옆 모니터에서 실시간 결과 확인
3. 멘티미터 스타일로 다른 사람들의 의견도 색상별 말풍선으로 흘러나옴

**기술 스택**:
- Vanilla JavaScript (프레임워크 없음)
- Firebase Realtime Database (실시간 동기화)
- 단일 `index.html` 파일 구조
- GitHub Pages 배포

**비주얼 컨셉**: 레드(#C8102E) + 화이트 — 전시회 공간 컨셉과 통일

## 🔀 3가지 모드 (URL 파라미터로 분기)

| URL | 모드 | 사용처 |
|---|---|---|
| `?mode=respond` | 응답 화면 | 부스 태블릿, 참여자 폰 (QR) |
| `?mode=display` | 실시간 결과 화면 | 부스 큰 모니터 |
| `?mode=admin` | 관리자 화면 (비밀번호) | 운영자 노트북/폰 |
| (mode 없음) | 모드 선택 메뉴 | — |

## 📋 설문 문항

1. **Q1** (필수, 단일선택): 가장 와닿는 아이디어 1개 — 5개 분과 카드 중 선택
2. **Q2** (필수, 자유서술 200자): 그 아이디어를 고른 이유
3. **Q3** (선택, 자유서술 300자): 활성화를 위한 추가 아이디어
4. **Q4** (필수, 별점 1-5): 전시·홍보 부스 만족도
5. **Q5** (선택, 자유서술 80자): 한줄평

## 💾 Firebase 데이터 구조

```
/responses/
  ├── {auto-id-1}/
  │   ├── timestamp: 1730xxxxxxxxx (ms)
  │   ├── division: "branding"
  │   ├── reason: "캐릭터가 귀여워서..."
  │   ├── idea: "다른 도시 콜라보도..."
  │   ├── satisfaction: 5
  │   ├── comment: "전시 잘 봤어요!"
  │   └── hidden: false
  └── {auto-id-2}/...
```

`hidden: true` 인 응답은 결과 화면·통계에서 제외됨 (관리자가 부적절한 응답 숨길 때 사용).

## ⚙️ 배포 전 필수 설정

### 1. Firebase Config 입력
`index.html` 상단의 `FIREBASE_CONFIG` 객체를 새 Firebase 프로젝트의 config로 교체:

```javascript
const FIREBASE_CONFIG = {
  apiKey: "...",
  authDomain: "...",
  databaseURL: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};
```

> ⚠️ 키즈 스케줄 프로젝트와는 **별개의 새 Firebase 프로젝트** 사용 권장 (데이터 격리)

### 2. Firebase Realtime Database 보안 규칙

테스트 모드(누구나 읽기/쓰기)는 30일 후 만료되므로, 행사 기간만 사용한다면 그대로 OK. 장기 운영 시:

```json
{
  "rules": {
    "responses": {
      ".read": true,
      ".write": true,
      "$id": {
        ".validate": "newData.hasChildren(['timestamp', 'division', 'reason', 'satisfaction'])"
      }
    }
  }
}
```

### 3. 관리자 비밀번호 변경
`index.html` 상단의 `ADMIN_PASSWORD` 값 수정.

### 4. GitHub Pages 배포
1. 새 저장소 생성 (예: `livinglab-survey`)
2. `index.html` 업로드
3. Settings → Pages → main 브랜치 활성화
4. `https://[username].github.io/livinglab-survey/` 접속

### 5. QR코드 만들기
응답 화면 URL(`https://.../livinglab-survey/?mode=respond`)을 QR로 만들어 부스에 비치.

## 🚀 부스 운영 시나리오

```
[큰 모니터]   ──→  ?mode=display     (계속 띄워둠, F11 풀스크린 권장)
                       ↑ 실시간 동기화
[태블릿 1]    ──→  ?mode=respond     (참여자가 직접 입력)
[태블릿 2]    ──→  ?mode=respond
[참여자 폰]   ──→  QR스캔 → ?mode=respond
[운영자 폰]   ──→  ?mode=admin       (CSV 다운로드, 응답 관리)
```

## 🧠 핵심 설계 원칙 (라라님 작업 패턴 기반)

1. **단일 HTML 파일 유지** — 별도 빌드 도구 없이 GitHub Pages에 그대로 배포
