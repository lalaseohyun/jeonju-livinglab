## 🎬 프로젝트 개요

**행사명**: 전주 영화의 거리 아이디어 제안소 (리빙랩 성과 전시·홍보 부스)

**목적**: 부스 방문객이 5개 분과의 제안 아이디어 중 가장 와닿는 것을 골라 의견을 남기고, 부스 모니터에서 실시간 결과·말풍선이 흐르며 라이브 반응을 보여주는 멘티미터 스타일 웹앱

**참여자 동선**:
1. 부스 방문 → QR 스캔 또는 태블릿으로 응답 (2~3분)
2. 응답 직후 옆 모니터에서 실시간 결과 + 다른 사람들의 의견 말풍선 자동 스크롤 확인

**기술 스택**:
- Vanilla JavaScript (프레임워크 없음)
- Firebase Realtime Database (실시간 동기화)
- 단일 `index.html` 파일 구조
- GitHub Pages 자동 배포

**비주얼 컨셉**: 레드(#C8102E) + 화이트 — 전시회 공간 컨셉과 통일

## 🔀 3가지 모드 (URL 파라미터 분기)

| URL | 모드 | 사용처 |
|---|---|---|
| `?mode=respond` | 응답 화면 (페이지네이션) | 부스 태블릿, 참여자 폰 (QR) |
| `?mode=display` | 실시간 결과 화면 (전체화면 토글) | 부스 큰 모니터/전광판 |
| `?mode=admin` | 관리자 화면 (비밀번호) | 운영자 노트북/폰 |
| (mode 없음) | 모드 선택 메뉴 | — |

### 응답 화면(`?mode=respond`) 페이지 구조

질문이 한 화면에 다 보이지 않고 **페이지 단위로 진행** — 인트로 + Q1~Q5로 총 6 페이지:

```
[인트로 페이지]  → "응답 시작하기"
       ↓
[Q1] 분과 선택 (필수)
       ↓
[Q2] 선택 이유 (필수, 200자)
       ↓
[Q3] 추가 아이디어 (선택, 300자)
       ↓
[Q4] 별점 (필수)
       ↓
[Q5] 한줄평 (선택, 80자) + 제출
       ↓
[감사 화면]
```

각 페이지에는 이전/다음 버튼, 진행률 바(`n/5`), 단계 인디케이터가 표시됨. 필수 항목 미입력 시 "다음" 버튼 비활성화. 페이지 이동 시 입력값/별점/분과 선택 상태는 모두 유지.

### 결과 화면(`?mode=display`) 핵심 기능

- **전체화면 토글 버튼** (`⛶ 전체화면`): 우측 상단 — 클릭 시 16:9 풀스크린 진입, 주소창·작업표시줄 모두 사라져 전광판 송출에 적합. ESC로 종료
- **자동 스크롤**: 참여자들의 목소리(말풍선) 영역이 천천히 자동 스크롤(약 60px/s) → 끝 도달 시 1.8초 멈춤 후 처음으로 부드럽게 복귀, 끝없이 반복
- **큰 폰트 + 강한 굵기**: 부스 방문객이 멀리서도 읽을 수 있도록 1200px+ 데스크톱과 풀스크린에서 모든 텍스트가 대형화/볼드화

### 관리자 화면(`?mode=admin`)

비밀번호 로그인(코드 상단 `ADMIN_PASSWORD`로 변경 가능, 현재 `2026`) 후:
- 총 응답수, 평균 만족도, 1위 분과, 아이디어 응답수 통계
- CSV 다운로드 (전체 응답)
- 결과화면 새 창 열기 / 공유 링크 복사
- 응답 목록(부적절한 응답을 "숨기기"로 결과 화면에서 제외 가능, 복구 가능)

## 💾 Firebase 데이터 구조

```
/responses/
  ├── {auto-id-1}/
  │   ├── timestamp: 1730xxxxxxxxx (ms)
  │   ├── division: "branding"
  │   ├── reason: "캐릭터가 귀여워서..."
  │   ├── idea: "다른 도시 콜라보도..."
  │   ├── satisfaction: 5
  │   ├── comment: "전시 잘 봤어요!"
  │   └── hidden: false
  └── {auto-id-2}/...
```

`hidden: true` 인 응답은 결과 화면·통계에서 제외됨 (관리자 모드에서 부적절한 응답 숨길 때 사용).

### Firebase Realtime Database 보안 규칙 (현재 적용)

```json
{
  "rules": {
    "responses": {
      ".read": true,
      ".write": true,
      "$id": {
        ".validate": "newData.hasChildren(['timestamp', 'division', 'reason', 'satisfaction'])"
      }
    }
  }
}
```

- `/responses` 경로 안에서만 읽기/쓰기 허용 (다른 경로 접근 차단)
- 응답 데이터에 `timestamp`, `division`, `reason`, `satisfaction` 필드가 있어야만 저장 (쓰레기 데이터 차단)
- 콘솔에서 "보안 규칙이 공개로 정의되어 있어..." 빨간 경고는 정상(`.read: true`이므로) — 무시 OK

### 행사 종료 후 (필수)

1. 관리자 모드에서 CSV 다운로드 → 응답 데이터 백업
2. Firebase 콘솔에서 보안 규칙을 `{"rules": {".read": false, ".write": false}}`로 잠금 → 행사 후 어뷰징 차단

## 📱 QR 코드

부스에 비치할 QR 코드 두 가지가 저장소에 포함되어 있음:

- `qr-respond.png` — **흑백** (1060×1060, 모든 폰에서 가장 안정적, 인쇄 권장)
- `qr-respond-red.png` — **비비드 레드** (#FF0000, 디자인용 보조)

### 다운로드 직링크

- https://github.com/lalaseohyun/jeonju-livinglab/raw/main/qr-respond.png
- https://github.com/lalaseohyun/jeonju-livinglab/raw/main/qr-respond-red.png

### 인쇄 사이즈 가이드

| 용도 | 권장 크기 |
|---|---|
| 부스 안내판 (1m+ 거리) | 15cm × 15cm 이상 |
| 테이블 위 안내문 (30~40cm) | 5~8cm |
| 명함/스티커 (10~15cm) | 2~3cm 이상 |

### URL 변경 시 재생성

```bash
python generate_qr.py
```

`generate_qr.py` 안의 `URL` 상수만 고치고 실행하면 두 PNG가 새로 만들어짐. error correction `H`(30% 손상까지 인식 가능)로 설정되어 있어 부스 환경에 강함.

### 인쇄 전 필수 점검

1. 라라님 폰으로 스캔 → `?mode=respond`로 이동 확인
2. 다른 사람 폰으로도 스캔 → 모든 환경에서 작동 확인
3. **인쇄본**으로도 한 번 스캔 (모니터에선 되는데 인쇄본은 안 되는 경우 있음)
